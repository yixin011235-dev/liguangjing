# 博客更新推送流程

> 简单三步，快速更新博客！

---

## 📝 快速更新（日常使用）

### 步骤 1：写/修改文章
在 `source/_posts/` 目录下创建或编辑 Markdown 文件

### 步骤 2：本地预览（可选）
```bash
npx hexo server
```
访问 http://localhost:4000 预览效果

### 步骤 3：推送到服务器
在博客根目录执行一键部署：

```powershell
.\deploy.bat
```

或者（如果更喜欢命令行）：

```bash
npx hexo clean
npx hexo generate
# 然后手动同步到服务器 /var/www/hexo/
```

## 🎯 推荐的更新流程

### 方式一：使用部署脚本（推荐）
```powershell

# 2. 执行部署
.\deploy.bat

# 3. 等待30秒，访问 https://liguangjing.top 查看效果
```

### 方式二：使用 Hexo 原生命令
```powershell
# 1. 生成静态文件
npx hexo generate

# 2. 部署到服务器（需要在 _config.yml 中配置 hexo-deployer-git）
npx hexo deploy
```

---

## 📋 部署脚本工作流程

`deploy.bat` 会自动完成：

1. ✅ 清理旧文件
2. ✅ 生成新的静态网站
3. ✅ 同步文件到服务器（使用 rsync）
4. ✅ 设置正确的文件权限
5. ✅ 重启 Nginx 服务
6. ✅ 显示成功信息

整个过程约 1-2 分钟 ⏱️

## 🆘 常见问题

### Q: 部署失败怎么办？
1. 检查网络连接
2. 确认 SSH 密钥配置
3. 查看错误提示
4. 联系服务器管理员

### Q: 如何回滚到之前的版本？
```bash
# 在 GitHub 上找到之前的提交
git log
git revert <commit-id>
```

### Q: 可以同时推送到 GitHub 吗？
可以！执行：
```bash
git add .
git commit -m "更新内容"
git push origin main
```
