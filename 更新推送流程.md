## 📋 部署脚本工作流程

`deploy.bat` 自动完成：

1. ✅ 清理旧文件
2. ✅ 生成新的静态网站
3. ✅ 同步文件到服务器（使用 rsync）
4. ✅ 设置正确的文件权限
5. ✅ 重启 Nginx 服务
6. ✅ 显示成功信息

整个过程约 1-2 分钟 ⏱️

## 🆘 常见问题

### Q: 部署失败怎么办？
1. 检查网络连接
2. 确认 SSH 密钥配置
3. 查看错误提示
4. 联系服务器管理员

### Q: 如何回滚到之前的版本？
```bash
# 在 GitHub 上找到之前的提交
git log
git revert <commit-id>
```

### Q: 可以同时推送到 GitHub 吗？
可以！执行：
```bash
git add .
git commit -m "更新内容"
git push origin main
```
