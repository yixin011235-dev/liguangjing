# 博客部署指南

## 📋 部署流程

### 准备工作

#### 1. 安装必要工具
在本地电脑上安装以下工具：
- **Git Bash** 或 **Windows Terminal**（支持 SSH 和 rsync）
- **rsync**（文件同步工具，Git Bash 已包含）

#### 2. 配置腾讯云安全组
1. 登录 [腾讯云控制台](https://console.cloud.tencent.com/)
2. 进入 **云服务器** → **安全组**
3. 点击你的服务器对应的安全组
4. 添加**入站规则**：

| 协议 | 端口 | 来源 | 策略 |
|------|------|------|------|
| HTTP | 80 | 0.0.0.0/0 | 允许 |
| HTTPS | 443 | 0.0.0.0/0 | 允许 |
| SSH | 22 | 你的IP | 允许 |

### 首次部署（只需执行一次）

#### 1. 上传证书到服务器
在 **PowerShell** 中执行：

```powershell
# 上传证书
scp d:\my-blog\liguangjing.top_nginx\liguangjing.top.key root@175.178.189.55:/etc/nginx/ssl/
scp d:\my-blog\liguangjing.top_nginx\liguangjing.top_bundle.crt root@175.178.189.55:/etc/nginx/ssl/

# 设置私钥权限
ssh root@175.178.189.55 "chmod 600 /etc/nginx/ssl/liguangjing.top.key"
```

#### 2. 配置服务器（SSH 连接到服务器后执行）

```bash
# 创建目录
mkdir -p /var/www/hexo /etc/nginx/ssl /etc/nginx/sites-available /etc/nginx/sites-enabled

# 创建 Nginx 配置文件
nano /etc/nginx/sites-available/liguangjing.top
```

粘贴以下配置：

```nginx
server {
    listen 80;
    server_name liguangjing.top www.liguangjing.top;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name liguangjing.top www.liguangjing.top;

    ssl_certificate /etc/nginx/ssl/liguangjing.top_bundle.crt;
    ssl_certificate_key /etc/nginx/ssl/liguangjing.top.key;
    
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;
    
    root /var/www/hexo;
    index index.html;

    location / {
        try_files $uri $uri/ =404;
    }

    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml;
}
```

启用配置：

```bash
# 创建软链接
ln -s /etc/nginx/sites-available/liguangjing.top /etc/nginx/sites-enabled/

# 测试并重启
nginx -t
systemctl restart nginx
```

### 日常部署

#### 方法一：使用部署脚本（推荐）

**Windows 用户**：
```powershell
# 在博客目录执行
cd d:\my-blog
.\deploy.bat
```

**Mac/Linux 用户**：
```bash
cd ~/my-blog
chmod +x deploy.sh
./deploy.sh
```

#### 方法二：手动部署

```bash
# 1. 本地生成博客
npx hexo clean
npx hexo generate

# 2. 同步到服务器
rsync -avz --delete -e "ssh -p 22" ./public/ root@175.178.189.55:/var/www/hexo/

# 3. 重启 Nginx
ssh root@175.178.189.55 "systemctl restart nginx"
```

### 验证部署

部署成功后，访问：
- **HTTP**: http://liguangjing.top （会自动跳转到 HTTPS）
- **HTTPS**: https://liguangjing.top

## 🔧 常用命令

### 服务器端

```bash
# 查看 Nginx 状态
systemctl status nginx

# 查看错误日志
tail -f /var/log/nginx/error.log

# 查看访问日志
tail -f /var/log/nginx/access.log

# 测试配置
nginx -t

# 重启 Nginx
systemctl restart nginx
```

### 本地端

```bash
# 本地预览
npx hexo server

# 清理并生成
npx hexo clean && npx hexo generate

# 部署到服务器
npx hexo deploy
```

## ❓ 常见问题

### Q: 部署后网站显示 502 错误？
A: 检查 Nginx 配置和 PHP-FPM 服务状态：
```bash
systemctl status nginx
systemctl status php-fpm
```

### Q: SSL 证书不生效？
A: 检查证书路径和 Nginx 配置：
```bash
# 检查证书文件是否存在
ls -la /etc/nginx/ssl/

# 检查 Nginx 配置
nginx -t
```

### Q: 如何更新博客？
A: 编辑 `source/_posts/` 中的文章，然后运行部署脚本即可。
